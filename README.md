#   Network Sniffer & Analyzer

A premium, interactive Python terminal utility for capturing, decoding, and analyzing network packets in real-time. Built using `scapy` and `rich`, the tool features a beautiful Dashboard TUI and a post-capture Interactive Analyzer shell.

---

##  Features

- **Live Packet Sniffing**: Capture live network traffic on any active network interface. Supports Berkeley Packet Filter (BPF) syntax (e.g. `tcp`, `udp`, `port 80`).
- **Traffic Simulation Mode**: Generates realistic simulated packets (ARP handshakes, DNS queries/responses, HTTP GET requests/responses, ICMP pings) in real-time. Extremely useful for testing, demonstrations, or running in restricted sandboxes.
- **PCAP File Reader**: Load, filter, and inspect `.pcap` capture files generated by this program or external utilities like Wireshark.
- **Sample PCAP Generator**: Create a dummy PCAP file populated with mock protocol scenarios to practice offline.
- **Beautiful TUI Dashboard**: Real-time updating console layout featuring:
  - Header with system/interface configuration.
  - Live protocol counter statistics.
  - Rolling packet log table.
- **Interactive Analyzer Shell**: Inspect captured packets after stopping the capture:
  - Scroll through all captured packets with full search and filtering capabilities.
  - Visual layer tree decomposition showing all header fields.
  - Side-by-side hex dump and ASCII payload view.
  - Save session captures to standard `.pcap` files.

##  Installation & Setup

### Prerequisites

1. **Python 3.7+**
2. **Npcap (for Windows)**: 
   To capture live packets on Windows, the Npcap library must be installed. Npcap provides the kernel-level driver interface for packet sniffing.
   - Download Npcap from the official website: [https://npcap.com/](https://npcap.com/).
   - During installation, make sure the option **"Install Npcap in WinPcap API-compatible Mode"** is selected (this is standard and ensures Scapy can find it).
   - *Note: If Npcap is already installed on your system, you are ready to go!*

### Installing Dependencies

Install the required Python packages:
```bash
pip install scapy rich
```

##  How to Run

### 1. Running in Live Capture Mode (Requires Administrator)

To capture actual live traffic flowing through your Wi-Fi or Ethernet card, you **must run your terminal as Administrator** (so that Python can open raw socket handles via the Npcap driver).

**On Windows:**
1. Search for **PowerShell** or **Command Prompt** in the Start Menu.
2. Right-click it and select **"Run as administrator"**.
3. Navigate to the project directory:
   ```powershell
   cd d:\phython\internship
   ```
4. Run the program:
   ```bash
   python sniffer.py
   ```
5. Select **Option 1 (Live Packet Sniffing)**, choose your active network adapter (e.g. Wi-Fi), set an optional filter, and press Enter!

### 2. Running in Simulation or PCAP Reader Mode (No Privileges Required)

If you don't have Administrator rights or want to test the tool offline, you can run the program normally:
```bash
python sniffer.py
```
- **Option 2 (Analyze PCAP File)**: Load any `.pcap` file to inspect.
- **Option 3 (Live Traffic Simulation)**: Generate mock live traffic to see the sniffer in action.
- **Option 4 (Generate Sample PCAP File)**: Creates a sample `sample.pcap` file in the directory that you can subsequently load and inspect.

## 🔍 Interactive Analyzer Commands

Once you stop the live capture or simulation by pressing **`Ctrl+C`**, the program opens the **Interactive Analyzer Shell** (`analyzer>>`). Use the following commands:

- `help` / `h` : List all available commands.
- `list` / `l` : Open a scrollable, searchable list of all captured packets (uses your system's terminal pager).
- `detail <id>` / `d <id>` : Deep-dive into a packet (e.g. `detail 3`). Generates a hierarchical protocol header tree and a formatted hex/ASCII dump.
- `filter <term>` / `f <term>` : Apply a display filter (e.g. `filter tcp`, `filter 8.8.8.8`, `filter GET`).
- `clear` / `c` : Clear the display filter.
- `save <filename>` / `s <filename>` : Save the captured session to a `.pcap` file (e.g. `save my_capture.pcap`).
- `resume` / `r` : Resume packet capture.
- `menu` / `m` : Exit the session and return to the main menu.
- `exit` / `q` : Quit the program.

##  Protocol Reference & Data Flow

When sniffing, you are observing data formatted according to the **OSI Model** (typically encapsulated in the TCP/IP stack). Each packet represents a nested onion of protocols:

```
┌────────────────────────────────────────────────────────┐
│ Ethernet Layer (MAC Addresses: src -> dst)             │
│   ┌──────────────────────────────────────────────────┐ │
│   │ IP Layer (IP Addresses: src -> dst, TTL, Proto)  │ │
│   │   ┌────────────────────────────────────────────┐ │ │
│   │   │ Transport Layer (TCP/UDP Ports, Flags)     │ │ │
│   │   │   ┌──────────────────────────────────────┐ │ │ │
│   │   │   │ Application Layer (HTTP, DNS, Raw)   │ │ │ │
│   │   │   └──────────────────────────────────────┘ │ │ │
│   │   └────────────────────────────────────────────┘ │ │
│   └──────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────┘
```

1. **Layer 2 (Link - Ethernet)**: Defines communication between local hardware adapters. Features MAC addresses (e.g. `8e:30:4b:d0:0b:5e`).
2. **Layer 3 (Network - IP)**: Routes packets across different networks using logical addresses. Features IP addresses (e.g. `192.168.1.10` to `8.8.8.8`).
3. **Layer 4 (Transport - TCP/UDP/ICMP)**:
   - **TCP**: Connection-oriented. Uses handshakes (`SYN`, `SYN-ACK`, `ACK`) to guarantee delivery. Features Sequence and Acknowledgment numbers.
   - **UDP**: Connectionless. Sends packets without handshakes (used by fast, low-overhead services like DNS).
   - **ICMP**: Diagnostic messages (e.g. ping requests and replies).
4. **Layer 7 (Application - HTTP/DNS)**: The actual data being consumed.
   - **HTTP**: Web browsing data (contains verbs like `GET` or `POST` and server response codes like `200 OK`).
   - **DNS**: Domain resolution (queries like `google.com` and replies mapping domains to IP addresses).
